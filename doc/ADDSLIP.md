# ADDSLIP（伝票登録）README

---

## 1. 概要

`add_slip` は、伝票の新規登録を行う画面である。

本画面の役割は以下の通り。

1. 伝票ヘッダ情報の入力（発注日・取引先・納品日など）
2. 明細（商品・数量・単価）の入力
3. 入力内容の検証
4. サーバーへ登録処理を実行

本READMEは以下を目的とする。

* 初学者でも理解できること
* AIや他開発者が同一機能を再現できること

---

## 2. 対象ファイル構成

本画面は以下の3ファイルで構成する。

* add_slip.cfm（画面）
* add_slip.js（フロント処理）
* add_slip.cfc（API）

---

## 3. 画面仕様

### 3.1 ヘッダ入力項目

| 項目  | 内容              |
| --- | --------------- |
| 発注日 | 日付入力（flatpickr） |
| 取引先 | モーダル検索で選択       |
| 納品日 | 日付入力            |
| メモ  | 任意入力            |

---

### 3.2 明細入力

テーブル形式で複数行入力可能

| 項目     | 内容             |
| ------ | -------------- |
| 商品コード  | 手入力（blurで自動取得） |
| JANコード | 自動入力           |
| 商品名    | 自動入力           |
| 数量     | 数値入力           |
| 単価     | 自動入力           |
| 削除     | 行削除            |

---

### 3.3 ボタン

| ボタン | 動作      |
| --- | ------- |
| 行追加 | 明細行を追加  |
| クリア | 入力内容初期化 |
| 登録  | データ登録   |

---

## 4. 主な処理フロー

### 4.1 登録処理

1. 必須チェック（CommonValidation）
2. 明細チェック
3. 確認ダイアログ表示
4. fetchでAPI呼び出し
5. 結果に応じてメッセージ表示

---

## 5. フロント処理（add_slip.js）

### 5.1 初期処理

* Enterキー移動制御
* テーブル操作バインド
* 日付ピッカー初期化
* 商品コード自動取得設定

---

### 5.2 明細処理

#### 明細収集

* `collectDetailList()`
* 空行を除外して配列化

#### 入力チェック

* `getDetailErrorMessage()`
* 行単位でエラーチェック

#### 合計計算

* `calculateTotals()`

---

### 5.3 商品自動取得

* 商品コード入力後 blurでAPI呼び出し
* JAN / 商品名 / 原価を取得
* エラー時はアラート表示

---

### 5.4 取引先モーダル

* 検索・一覧表示
* 選択でフォーム反映

---

### 5.5 登録処理

```javascript
fetch("add_slip.cfc?method=saveSlip&returnformat=json")
```

送信データ：

```json
{
  "slip_date": "",
  "supplier_code": "",
  "supplier_name": "",
  "delivery_date": "",
  "memo": "",
  "total_qty": 0,
  "total_amount": 0,
  "detail_list": []
}
```

---

## 6. API仕様（add_slip.cfc）

### 6.1 saveSlip

#### 処理内容

* 伝票ヘッダ登録（t_slip）
* 明細登録（t_slip_detail）

#### 戻り値

```json
{
  "status": 0,
  "message": "登録成功"
}
```

---

### 6.2 getItemByCode

#### 処理内容

* 商品コードから商品情報取得

#### 戻り値

```json
{
  "status": 0,
  "results": {
    "jan_code": "",
    "item_name": "",
    "gentanka": 0
  }
}
```

## 7. バリデーション

### 7.1 ヘッダ

* 必須チェック

### 7.2 明細

* 商品コード必須
* 商品名必須
* 数量 > 0
* 単価 >= 0

---

## 8. UX仕様

### 8.1 Enterキー移動

* 入力項目間をフォーカス移動
* submitは発生しない

---

### 8.2 ローディング制御

* 登録中ボタン無効化
* 文言変更（登録中...）

---

### 8.3 エラーメッセージ

* SweetAlert2使用
* 行番号付きで表示

---

## 9. 注意点

* 明細は1行以上必須
* 数値はカンマ除去して計算
* HTMLはescapeして表示
* Enterでsubmitしない制御あり

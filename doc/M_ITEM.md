# 商品マスタ一覧（m_item）

---

## 1. 概要

商品マスタの一覧表示・検索・ソート・ページング・新規登録遷移・詳細遷移・CSVエクスポートを行う画面。

本画面では以下を実現する。

* 商品情報の検索・絞り込み
* 商品一覧の表示
* 商品詳細画面への遷移
* 新規登録画面への遷移
* CSVエクスポート
* 一覧状態の保持

---

## 2. 対応ファイル

| 種別    | ファイル名             |
| ----- | ----------------- |
| 画面    | m_item.cfm        |
| API   | m_item.cfc        |
| JS    | m_item.js         |
| CSV出力 | m_item_export.cfm |

---

## 3. 機能一覧

### 3.1 検索機能

以下の条件で検索可能。

* 商品コード
* JANコード
* 商品名

検索仕様は以下の通り。

* 商品コード：完全一致
* JANコード：完全一致
* 商品名：部分一致
* 商品名カナも商品名検索の対象に含める

---

### 3.2 Enterキーによるフォーカス移動

検索欄では Enter キー押下時に検索を即実行せず、次の入力欄へ移動する。

* Enter：次の検索項目へ移動
* Shift + Enter：前の検索項目へ移動
* 最後の項目では検索ボタンへフォーカス
* IME変換中は無視する

---

### 3.3 一覧表示

一覧の表示項目は以下。

* 商品コード
* JANコード
* 商品名
* 作成日時
* 更新日時

一覧の各行はクリック可能であり、対象商品の詳細画面へ遷移する。

---

### 3.4 詳細画面遷移

一覧の行クリック時、POST送信で `m_item_dt.cfm` へ遷移する。

遷移時の表示モード：

* `display_mode = view`

合わせて以下の一覧状態を引き継ぐ。

* 検索条件
* ソート条件
* 現在ページ

---

### 3.5 新規登録遷移

ヘッダの追加ボタン押下時、POST送信で `m_item_dt.cfm` へ遷移する。

遷移時の表示モード：

* `display_mode = add`

新規登録遷移時にも一覧状態を保持する。

---

### 3.6 ソート機能

ソートボタン押下でソート状態を切り替える。

状態遷移は以下。

未指定 → 昇順 → 降順 → 未指定

対象項目：

* 商品コード
* JANコード
* 商品名
* 作成日時
* 更新日時

ソート状態はアイコンで表示する。

---

### 3.7 ページング

1ページあたり 50 件表示。

ページ移動ボタン：

* 最初へ
* 前へ
* 次へ
* 最後へ

現在ページと総ページ数を表示する。

---

### 3.8 CSVエクスポート

ヘッダのエクスポートボタン押下時、現在の検索条件とソート条件を付けて `m_item_export.cfm` を呼び出し、CSVを出力する。

引き継ぐ条件：

* search_product_code
* search_jan_code
* search_product_name
* sortField
* sortOrder

---

## 4. API仕様

### 4.1 一覧取得API

API：

`m_item.cfc?method=getItemList&returnformat=json`

#### リクエスト

```json
{
  "page": 1,
  "pageSize": 50,
  "sortField": "",
  "sortOrder": "",
  "search_product_code": "",
  "search_jan_code": "",
  "search_product_name": ""
}
```

#### レスポンス

```json
{
  "status": 0,
  "message": "商品一覧を取得しました。",
  "results": [
    {
      "item_code": "ITEM001",
      "jan_code": "4900000000001",
      "item_name": "商品A",
      "create_datetime": "2026/04/03 09:00:00",
      "update_datetime": "2026/04/03 10:00:00"
    }
  ],
  "paging": {
    "page": 1,
    "pageSize": 50,
    "totalCount": 1,
    "totalPage": 1,
    "hasPrev": false,
    "hasNext": false
  }
}
```

---

### 4.2 ステータス仕様

| status | 意味      |
| ------ | ------- |
| 0      | 正常      |
| 1      | 異常（エラー） |

本画面の一覧取得APIは、このルールで扱う。

---

## 5. 検索仕様

### 5.1 商品コード

* `item_code = 入力値`
* 完全一致で検索

### 5.2 JANコード

* `jan_code = 入力値`
* 完全一致で検索

### 5.3 商品名

以下のいずれかに部分一致した場合に該当とする。

* item_name
* item_name_kana

---

## 6. ソート仕様

ソート項目に応じて、以下のカラムで並び替える。

| sortField       | 並び替え対象 |
| --------------- | ------ |
| item_code       | 商品コード  |
| jan_code        | JANコード |
| item_name       | 商品名    |
| create_datetime | 作成日時   |
| update_datetime | 更新日時   |

`sortOrder = desc` の場合は降順、それ以外は昇順とする。

未指定時の基本並び順：

* 商品コード 昇順

---

## 7. 一覧表示仕様

### 7.1 データなし

検索結果が0件の場合は、一覧テーブルに以下を表示する。

* データがありません。

### 7.2 読み込み中

読み込み中は、一覧テーブルに以下を表示する。

* 読み込み中です...

### 7.3 エラー時

API異常または通信失敗時は、一覧テーブルにエラーメッセージを表示する。

---

## 8. 状態保持

一覧から詳細へ遷移し、詳細から一覧へ戻った際に以下を保持する。

* 検索条件
* ソート条件
* 現在ページ

保持対象：

* return_search_product_code
* return_search_jan_code
* return_search_product_name
* return_sort_field
* return_sort_order
* return_page

hidden項目に保存して管理する。

## 10. エクスポート仕様

### 10.1 出力ファイル名

出力ファイル名は以下の形式。

`商品マスタ_yyyymmdd_HHmmss.csv`

例：

`商品マスタ_20260403_103015.csv`

---

### 10.2 出力条件

一覧画面の検索条件・ソート条件をそのまま引き継いで出力する。

対象条件：

* 商品コード
* JANコード
* 商品名
* ソート項目
* ソート順

---

### 10.3 出力項目

CSVのヘッダは以下。

* 商品コード
* JANコード
* 商品名
* 商品名カナ
* 原価
* 売価
* 分類
* 使用区分
* 作成日時
* 作成者コード
* 作成者名
* 更新日時
* 更新者コード
* 更新者名

---

### 10.4 出力内容の変換仕様

#### 分類（item_category）

| 値 | 出力文字列 |
| - | ----- |
| 1 | 食品    |
| 2 | 雑貨    |
| 3 | 日用品   |
| 4 | 衣料    |
| 5 | 小物    |

上記以外は空文字とする。

#### 使用区分（use_flag）

| 値   | 出力文字列 |
| --- | ----- |
| 1   | 有効    |
| その他 | 無効    |

---

### 10.5 日時出力形式

以下の形式で出力する。

`YYYY/MM/DD HH:mm:ss`

対象：

* 作成日時
* 更新日時

---

### 10.6 NULL値の扱い

NULL値は空文字で出力する。

対象例：

* JANコード
* 商品名カナ
* 原価
* 売価
* 作成者コード
* 更新者名

---

### 10.7 出力処理の流れ

1. URLパラメータから検索条件・ソート条件を取得
2. ORDER BY句を組み立てる
3. WHERE句を組み立てる
4. 一時フォルダにCSVファイルを作成
5. ヘッダ行を先に1回だけ書き込む
6. データ本体を `export_db.cfc` 経由で追記する
7. `download.cfm` へリダイレクトしてダウンロードさせる

---

## 11. エラーハンドリング

### 11.1 一覧取得API

例外発生時：

* ログ出力
* `status = 1`
* メッセージ返却

メッセージ：

* 商品一覧の取得中にエラーが発生しました。

### 11.2 画面側

* 通信失敗時はコンソールにエラーを出力
* 一覧テーブルにはエラーメッセージ行を表示

### 11.3 CSV出力

エクスポート処理でエラー文字列が返った場合は、テキストとしてエラーメッセージを出力して中断する。

---

## 12. 注意点

* 正常時は `status = 0`
* 異常時は `status = 1`
* 商品名検索は商品名カナも対象
* Enterキーでは即検索せず、入力移動を優先
* 読み込み中は操作不可
* ソート未指定時は商品コード昇順
* 一覧とエクスポートで検索条件・ソート条件を共通化している

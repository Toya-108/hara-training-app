# マスタ共通仕様（MASTER）

---

## 1. 概要

本ドキュメントは、本システムにおける各マスタ機能の共通仕様を定義する。

対象マスタ：

* 商品マスタ（m_item）
* 取引先マスタ（m_supplier）
* 社員マスタ（m_staff）
* 配送業者マスタ（m_delivery_company）

---

## 2. 共通構成

各マスタは以下の構成で統一する。

| 種別  | ファイル                 |
| --- | -------------------- |
| 一覧  | xxx.cfm              |
| 詳細  | xxx_dt.cfm           |
| API | xxx.cfc / xxx_dt.cfc |
| JS  | xxx.js / xxx_dt.js   |
| CSV | xxx_export.cfm       |

---

## 3. 画面構成

### 3.1 一覧画面

* 検索条件入力
* 一覧テーブル
* ページング
* ヘッダ操作（追加・エクスポート）

---

### 3.2 詳細画面

* 入力フォーム
* モード切替（view / edit / add）
* 保存・削除操作

---

## 4. 画面モード共通仕様

| モード  | 内容   |
| ---- | ---- |
| view | 表示専用 |
| edit | 編集   |
| add  | 新規登録 |

制御方法：

* `display_mode` により制御
* JavaScriptでUI状態を切替

---

## 5. API共通仕様

### 5.1 ステータス

| status | 意味      |
| ------ | ------- |
| 0      | 正常      |
| 1      | 異常（エラー） |

---

### 5.2 レスポンス形式

すべてのAPIは以下形式で統一する。

```json
{
  "status": 0,
  "message": "",
  "results": {},
  "paging": {}
}
```

---

### 5.3 設計ルール

* レスポンスキーはすべて小文字で統一
* 正常判定は `status === 0`
* エラー時は `message` を必ず返却する

---

## 6. 一覧画面共通仕様

### 6.1 検索

* 各マスタごとに検索条件を持つ
* フォーム入力値をそのままAPIに送信

---

### 6.2 Enterキー挙動（統一仕様）

すべてのマスタで以下に統一する。

* Enterキー押下 → 次の入力項目へフォーカス移動
* Shift + Enter → 前の入力項目へ移動
* 最後の項目 → 検索ボタンへフォーカス
* IME変換中は無視
* Enterで検索は実行しない

---

### 6.3 ソート

状態遷移：

未指定 → 昇順 → 降順 → 未指定

---

### 6.4 ページング

* 基本：50件 / ページ
* 在庫画面：10件 / ページ
* 前後移動あり
* ページ番号表示あり

---

### 6.5 状態保持

以下を保持する。

* 検索条件
* ソート条件
* ページ番号

hidden項目で管理する。

---

### 6.6 詳細遷移

* POST送信で遷移
* `display_mode = view`
* 一覧状態を引き継ぐ

---

### 6.7 新規登録

* `display_mode = add` で遷移

---

### 6.8 エクスポート

* 検索条件・ソート条件を引き継ぐ
* CSV形式で出力

---

## 7. 詳細画面共通仕様

### 7.1 保存処理

処理フロー：

1. 入力チェック
2. 確認ダイアログ表示
3. API実行
4. viewモードへ遷移

---

### 7.2 削除処理

* viewモード時のみ実行可能
* 確認ダイアログを表示

---

### 7.3 戻り処理

* 一覧または元画面へ戻る
* hidden項目で遷移元を管理

---

### 7.4 モード制御

```javascript
function applyModeLayout()
```

---

## 8. バリデーション共通仕様

### 8.1 必須チェック

* コード系項目
* 名称系項目

---

### 8.2 エラー表示

* SweetAlertで表示
* エラー項目へフォーカス

---

### 8.3 数値チェック

* 数値項目は数値のみ許可
* 不正入力時は値をクリア

---

## 9. 表示変換共通仕様

### 9.1 使用区分

| 値   | 表示       |
| --- | -------- |
| 1   | 有効 / 使用中 |
| その他 | 無効 / 停止  |

---

### 9.2 権限区分

| 値   | 表示  |
| --- | --- |
| 9   | 管理者 |
| 2   | 責任者 |
| その他 | 一般  |

---

### 9.3 商品分類

| 値 | 表示  |
| - | --- |
| 1 | 食品  |
| 2 | 雑貨  |
| 3 | 日用品 |
| 4 | 衣料  |
| 5 | 小物  |

---

### 9.4 数値表示

* カンマ区切り表示（toLocaleString）

---

## 10. エラーハンドリング

### API

* `status !== 0` の場合はエラー
* messageを画面に表示

---

### 通信エラー

* console.errorでログ出力
* アラートまたはダイアログ表示

---

## 11. セキュリティ

* HTMLエスケープ実施
* SQLはバインド変数使用
* パスワードはハッシュ化（社員マスタ）
